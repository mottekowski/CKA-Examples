# Analysis of the orig-sock-shop Kubernetes Deployments

## 1. Carts Service (01-carts-dep.yaml)
- **Deployment Name**: carts
- **Namespace**: sock-shop
- **Replicas**: 1
- **Container Image**: weaveworksdemos/carts:0.4.8
- **Ports**: 80
- **Environment Variables**: JAVA_OPTS for Java configuration.
- **Resource Limits**: CPU: 300m, Memory: 500Mi
- **Security Context**: Non-root user, read-only root filesystem.

## 2. Carts Database (03-carts-db-dep.yaml)
- **Deployment Name**: carts-db
- **Namespace**: sock-shop
- **Replicas**: 1
- **Container Image**: mongo
- **Ports**: 27017
- **Environment Variables**: MYSQL_ROOT_PASSWORD and MYSQL_DATABASE.
- **Security Context**: Limited capabilities, read-only root filesystem.

## 3. Catalogue Service (05-catalogue-dep.yaml)
- **Deployment Name**: catalogue
- **Namespace**: sock-shop
- **Replicas**: 1
- **Container Image**: weaveworksdemos/catalogue:0.3.5
- **Ports**: 80
- **Environment Variables**: Command and arguments for the application.
- **Resource Limits**: CPU: 200m, Memory: 200Mi
- **Liveness and Readiness Probes**: Health checks on /health endpoint.

## 4. Catalogue Database (07-catalogue-db-dep.yaml)

## 5. Front-End Service (09-front-end-dep.yaml)
